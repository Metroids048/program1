# 桌面端 UI 交付闸口

## 适用范围

- 首页岗位台
- 实时助手
- 模拟面试
- JD 分析
- 问题库
- 简历
- 记录复盘

## 首屏约束

### 首页

- 首屏只保留页面标题、岗位卡、主 CTA、最小上下文。
- 禁止说明型大字标题。
- 禁止首页内 `JD 分析 / 问题库 / 简历` 快捷入口。
- 禁止重复展示当前岗位名的底部信息条。

### 实时助手

- 顶部只允许单行状态栏：岗位、题号、计时器、AI 状态、结束。
- 左侧只允许语音按钮、识别文本、可编辑文本、主 CTA。
- 禁止出现 `interim / final` 等技术标签。
- 禁止出现 `查看 JD 摘要 / 去模拟练习 / 去 JD 分析`。
- 禁止常驻“保存记录”按钮。

### 模拟面试

- 顶部只允许进度、题号、倒计时、面试官风格 badge、结束。
- 右侧只保留提词卡。
- 面试官风格选择必须放入进入前配置弹窗。
- 禁止 `当前均分 / 连续高分 / 已答` 顶部指标。
- 禁止 `朗读 / 暂停 / 我说完了` 常驻按钮。

### JD 分析

- 左栏默认先展示简历差异。
- 关键词超过 6 个默认折叠。
- JD 原文默认折叠。
- 分析对话默认收起，以抽屉形式出现。

### 问题库

- 分类必须以 Tab 或显式分组显示。
- 单题默认折叠。
- 默认列表态只展示题干、难度、高频/状态。
- 上传文件与新增题目必须在页头高优先级区域。

### 简历

- 保持三栏，但中栏只允许展示当前选中模块的完整内容。
- 左栏仅导航和上传入口。
- 右栏只保留当前模块优化和匹配分析。
- 禁止中栏同时平铺多个大模块卡片。

### 记录复盘

- 空状态只保留一个 CTA。
- 筛选器并排、宽度自适应。
- 列表态必须展示模式、岗位、日期、摘要、分数或题数。

## 开发前输入

每次桌面端 UI 重构必须先固定 3 份输入：

1. 页面级线框或截图参考。
2. 每页“首屏保留项 / 禁止项”清单。
3. 验收截图位和测试断言。

## 验收要求

- `npm run verify`
- Browser 冒烟
- 控制台无 error
- 测试断言覆盖“错误结构不能再出现”
