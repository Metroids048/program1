# AI 求职台｜产品分析 × UI 重设计全方案

> 基于 GitHub 仓库 `Metroids048/program1` 完整读取后输出，供前端重构直接使用。

---

## 第一部分：产品现状分析

### 项目核心定位
面向应届生/校招用户的 AI 面试辅助工具，核心主线为：
**导入简历与 JD → 开启实时面试助手 → 识别问题 → 生成提词卡 → 保存记录 → 复盘改进**

对标竞品：Final Round AI、LockedIn AI、Interviews.chat、OfferGoose。

---

### 优势（继续保持）

**功能闭环完整。** 从 JD 导入 → 实时助手 → 模拟练习 → 复盘报告，链路跑通，是同类产品中覆盖面广的 MVP。

**诚实的 AI 状态设计。** 明确区分「模型生成」与「本地练习模式」，不伪装 fallback，这一点比很多竞品更有用户诚信度。

**本地优先架构。** SQLite + 本地存储，无需账号即可上手，对校招用户友好。

**双模式并存。** 实时助手（面试中用）+ 模拟练习（面试前练），适配不同使用场景。

**语音交互边界清晰。** 只用麦克风，不抓系统音频，合规且在面试场景中可实际使用。

---

### 劣势（本次重点优化）

**视觉层极度混乱。** AGENTS.md 自述「避免堆字、卡片套卡片、首屏塞满诊断信息」，说明团队已意识到问题但代码未落地。

**信息层级缺失。** 字号体系未建立，标题、正文、标注、状态文字混排，用户无法快速抓取核心信息。

**首页目标不明。** 大量 JD 诊断信息铺满首屏，核心动作「开启面试助手」被埋没。对标 Final Round AI 首页：进入即是对话框，一眼知道该做什么。

**功能页边界模糊。** home / live / context / records 四个组件承载过多，每页都想说太多，反而哪个场景都说不清楚。

**缺乏视觉设计语言。** 无统一色彩系统、无字体规范、无间距基准，导致每次迭代都在破坏已有一致性。

**移动端适配缺位。** AGENTS.md 指出 390px 要求，但实际布局未为移动端优化。

---

### 下一步产品优化方向（不增加新功能）

1. 重建视觉设计系统（色彩、字体、间距 Token）
2. 首页聚焦：第一屏只做一件事——让用户开始面试
3. 每个功能页做减法，突出当前页的核心操作
4. 建立清晰的状态语言（AI 生成中 / 本地模式 / 已保存 / 错误）
5. 提词卡组件标准化，成为产品最有辨识度的 UI 元素

### P0｜应用导航框架

**设计原则：** 导航极简，当前位置清晰，不占用内容区高度。

**结构：**
```
┌─────────────────────────────────────────────────────────┐
│  🎯 AI 求职台     [首页] [助手] [练习] [复盘]    [设置]  │  ← 顶部导航，高度 56px
└─────────────────────────────────────────────────────────┘
│                                                         │
│                    主内容区                              │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

**导航 Tab 命名（精简）：**
- 首页（岗位）→ 实时助手 → 模拟练习 → 复盘记录

**UI 要求：**
- Tab 激活状态：primary 蓝色下划线，文字加粗
- 未激活：灰色文字，无背景
- 移动端：底部 Tab Bar 替代顶部导航

**页面唯一目标：** 让用户选好当前要面的岗位，然后开始。

**信息层级（从上到下）：**

```
│  │   🎙️ 开始面试助手  │  │   🧪 开始模拟练习  │        │  ← 主 CTA，等高双列
│  │   实时面试中使用   │  │   面试前热身练习   │        │
│  └────────────────────┘  └────────────────────┘        │
│                                                         │
│  面试配置                                  [编辑]        │

│                                                         │

└─────────────────────────────────────────────────────────┘
```

**产品设计要点：**
- 岗位卡只展示「当前激活岗位」，切换岗位收进下拉 Sheet，不在首页展开多个岗位
- 两个 CTA 按钮等权重，用图标+标题+一句说明三层结构区分场景
- 面试配置折叠展示，只显示关键值，点编辑才展开
- 最近记录最多 2 条，入口引导去复盘页

**不放在首页的内容：**
- JD 全文
- 简历评分详情
- 问题库列表
- 技能图谱

### P3｜实时助手页（面试中核心页面）

**页面目标：** 面试进行中，快速识别问题，生成提词卡。

**布局：左右分栏（桌面）/ 单列（移动端）**

```
┌──────────────────────┬──────────────────────────────────┐
│  听取 / 输入问题      │  当前提词卡                       │
│                      │                                  │
│  ┌──────────────────┐│  ┌──────────────────────────────┐│
│  │ [●] 正在听取...  ││  │ 🤖 AI 生成                   ││
│  │                  ││  │                              ││
│  │ "请介绍一下你在  ││  │ 问题类型：行为面试             ││
│  │  上一段实习中最  ││  │                              ││
│  │  有挑战的项目。" ││  │ 开场句                        ││
│  │                  ││  │ 「在字节实习期间，我负责了…」  ││
│  └──────────────────┘│  │                              ││
│                      │  │ 核心要点（STAR）               ││
│  [🎙停止] [✏️手动输入]│  │ S: 用户增长停滞，次留率低     ││
│                      │  │ T: 独立设计留存策略            ││
│  ─────────────────── │  │ A: 数据分析 → 功能迭代        ││
│                      │  │ R: 次留 +12%，月活回正         ││
│  状态                │  │                              ││
│  后端已连接 ✓         │  │ 可引用证据                    ││
│  模型: DeepSeek      │  │ 「DAU 数据、A/B 实验报告」     ││
│                      │  │                              ││
│                      │  │ 风险提醒                      ││
│                      │  │ 避免过度强调个人，注意团队协作  ││
│                      │  │                              ││
│                      │  │ 可能追问                      ││
│                      │  │ 「你的数据来源是什么？」        ││
│                      │  └──────────────────────────────┘│
│                      │                                  │
│                      │  [📋 保存这张卡] [🔄 重新生成]   │
└──────────────────────┴──────────────────────────────────┘
```

**UI 设计要点：**
- 提词卡是这个产品最高价值的输出，视觉上要最突出
- 提词卡结构固定：问题类型 → 开场句 → 核心要点 → 证据 → 风险 → 追问
- AI 生成 vs 本地练习用颜色明确区分：AI 生成紫色标签，本地模式灰色标签
- 听取文字区域：interim（半透明灰）/ final（正常黑色）/ 可编辑状态有光标
- 停止听取后文字保留，不自动清空
- 手动输入 fallback：文字输入框 + 「生成提词卡」按钮

**移动端适配：**
```
┌─────────────────────┐
│  [●] 正在听取       │
│  "请介绍一下…"       │
│  [停止]  [手动输入] │
├─────────────────────┤
│  提词卡             │
│  问题类型：行为面试  │
│  开场句：「…」       │
│  核心要点 ···       │
│  [展开全部 ↓]       │
└─────────────────────┘
```

```
│  A  数据分析 → 方案设计 → A/B 验证                      │
│  R  次留 +12%，月活恢复正增长                           │
├─────────────────────────────────────────────────────────┤
```
