## 覆盖的真实用户流程

- 注册新用户，获取登录 token，并进入 onboarding
- 导入 PDF 简历，写入 profile
- 粘贴真实 JD 创建岗位，生成岗位上下文
- 上传 DOCX 项目资料到当前岗位
- 手动添加面试问题到当前岗位
- 实时助手生成 SSE 提词卡，包含 stage / delta / card / done
- 模拟面试创建 session，回答问题，生成追问、报告维度和 conversationHistory
- 保存面试记录，列表与详情可读取
- 导出数据，非法导入拒绝，round-trip 导入保持 records
- 服务重启后 state / records 保持一致
- AI 在线模型覆盖 intake、cue-card、mock session、mock answer、resume AI
- RAG 覆盖资料、问题、岗位上下文、evidenceTrace、retrievalCount
- 语音分析覆盖语速、口头禅、0 秒时长和 Node 环境无浏览器语音 API

## 发现与处理

本轮最初两个专项脚本出现失败，根因不是产品链路坏掉，而是验收脚本仍按旧匿名共享桶口径调用 owner-scoped 接口：

- `/api/positions/:id/materials`
- `/api/positions/:id/questions`
- `/api/positions/:id/context`
- `/api/records`
- `/api/import`
- `/api/rag/reindex`

已将 `scripts/codex-audit.ts` 与 `scripts/full-verify-ai-voice-rag.ts` 改为真实登录用户口径：先注册用户获取 token，再携带 `Authorization` 调用 owner-scoped 链路。复跑后：

- `scripts/codex-audit.ts`：40/40 Pass
- `scripts/full-verify-ai-voice-rag.ts`：32/32 Pass

同时将空回答、空问题的验收预期修正为“后端 400 拒绝空输入”，避免把正常校验误判为失败。
