## 2. 只验证了 fallback 的路径

下列路径在仓库自带测试中主要验证的是“本地练习模式可用”，不是“真实模型成功态”：

1. `/api/copilot/cue-card/stream`
2. `/api/mock/session/:id/answer`
3. `/api/resume/ai`
4. `/api/search`

说明：

- 这些路径在本轮真实模型临时回归中已额外证明前三条可进入 `success`
- 但仓库内常规自动化仍以 fallback 为主

## 3. 完全缺少自动化覆盖的关键路径

1. 账户导入失败时的错误提示与状态一致性
2. `/api/import` 失败后前端是否仍显示成功
3. `/api/search` 成功态
4. `cue-card` 真实模型输出的真实性约束
5. 记录页指标、Transcript、时间轴的完整语义正确性
6. 简历页“应用到整份编辑区”后的服务端同步一致性
7. 首页保存岗位后再编辑确认字段的完整回写
8. 问题库上传多个真实文件后的重复/覆盖行为

## 4. 浏览器层覆盖缺口

由于 Codex 限制，本轮完全未做真实浏览器渲染验收：

1. 首屏布局与视觉密度
2. 移动端排版
3. 控制台运行时错误
4. 真实滚动、输入法、焦点、textarea 自增高等细节
5. SSE 在真实浏览器中的长连接稳定性

## 5. 测试资产缺口

1. `scripts/full-flow-retest.ts` 没有 npm script 入口
2. 没有统一的“真实模型成功态回归”脚本
3. 没有 `/api/search` 成功态测试夹具
4. 没有对“真实性 guardrail”做断言的测试
5. 没有覆盖账户导入导出异常路径的测试

## 6. 建议优先补的自动化

优先级建议：

### P1

1. 账户导入失败时不得提示成功
2. `cue-card` 输出不得包含“虚构 / 编造 / 假设项目”类建议
3. 真实模型成功态 smoke script 固化为 npm script

### P2

1. `/api/search` 成功态测试
2. 记录页语义断言
3. 问题库多文件导入与去重行为

### P3

1. 构建 chunk 体积告警门禁
2. lint warning 清零
