## 三、本轮发现并已修复的问题

| 编号 | 问题 | 严重度 | 处理 |
|------|------|--------|------|
| F1 | 法务/关于/帮助页有内容但**无路由、无入口**（注册同意、页脚、账户页均缺链接）—— C 端上线硬性合规缺口 | P0 | A1：新增 `/terms-of-service` `/privacy-policy` `/about` `/help` 路由 + 注册同意链接 + 全局页脚 + 账户页政策链接 |
| F2 | **游客数据隔离缺陷**：所有游客共享 `guest:preview` ownerKey，多游客并发会串数据/串 RAG 召回 | P0 | A3：按会话级 `x-guest-id` 隔离 ownerKey 与数据分区；新增游客互隔离回归测试 |
| F3 | **DeepSeek 调用无超时/无重试**，网络抖动直接降级 | P1 | A4：45s 超时 + 一次指数退避重试（5xx/网络/超时可重试，4xx 不重试），仍失败再走既有本地兜底 |
| F4 | 网络失败缺少**统一用户提示**，各处静默失败 | P1 | A2：新增全局 Toast，`apiFetch` 网络层失败统一提示 |
| F5 | 提词卡缺少**用户认可度采集**，AI 质量评估缺上游信号 | P2 | A5：cue card 加 👍/👎，复用 `/api/feedback` 记录 rating + skill |
