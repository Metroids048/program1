# 当前信息架构与页面职责

更新时间：2026-06-19

## 页面结构

- `/`
  - 首页
  - 职责：真实 JD intake 与岗位切换入口

- `/live`
  - 实时助手
  - 职责：接收面试问题，生成提词卡，保存实时记录

- `/mock`
  - 模拟面试
  - 职责：围绕当前岗位做预演、追问与练习记录

- `/jd`
  - JD 分析
  - 职责：汇总当前岗位的准备重点、可能问题、资料命中与差距

- `/questions`
  - 问题库
  - 职责：作为“当前岗位资料底座”，管理项目资料、上传资料和用户问题

- `/resume`
  - 简历
  - 职责：全局简历编辑与 AI 对话优化，不绑定单个岗位持久化，但会读取当前岗位做匹配分析

- `/records`
  - 面试记录
  - 职责：查看实时助手与模拟面试记录、报告与后续动作

## 当前首页职责边界

- 首页只做 intake，不做大段诊断展示。
- 首页输出的是“审核卡”，不是伪造出来的一整份岗位文案。
- 首页仍可进入模拟面试，但前提是岗位信息经过基本确认。

## 当前问题库职责边界

- 优先级固定：
  1. 项目资料区
  2. 上传资料区
  3. 用户问题库区

- 问题库不是训练计划页，不和上传资料混成一层。

## 当前简历页职责边界

- 保持三列布局，不改成单页合并产品。
- 右侧是标准 AI 对话，不是悬浮批注区。
- 保存行为走后端，不以前端全量状态为准。
