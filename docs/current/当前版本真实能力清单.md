# 当前版本真实能力清单

更新时间：2026-06-19

## 版本目标

当前代码目标是“公开 MVP 收口版”。

本轮已锁定：

- 保留现有多页面 IA，不合并页面。
- 首页改为真实 JD intake，不再伪造完整岗位草稿。
- 数据 owner 以后端为主，前端只保留临时输入态、离线快照缓存和 UI 偏好。
- 问题库重点是当前岗位的项目资料、上传资料、用户保存问题。
- 简历页右侧是正常 AI 对话面板，不是零散建议块。

## 当前真实能力

- 首页 `/`
  - 支持粘贴真实 JD 或岗位补充信息。
  - 审核卡分开展示 `用户原文`、`系统推断/用户确认`、`缺失字段`。
  - 字段来源明确标记为 `原文` / `系统推断` / `用户确认`。
  - 岗位列表只展示最小信息：岗位名、公司、更新时间、准备状态。

- 实时助手 `/live`
  - 保留实时问题输入、提词卡生成和记录保存。
  - 模型不可用时会标记本地练习模式，不伪装成模型成功。

- 模拟面试 `/mock`
  - 保留按岗位进行模拟提问、追问与记录保存。
  - 记录与报告以后端数据为准。

- JD 分析 `/jd`
  - 继续围绕当前岗位汇总准备重点、可能问题、资料命中和简历差距。

- 问题库 `/questions`
  - 当前岗位维度管理项目资料、上传资料、用户问题。
  - 默认优先展示用户手动保存问题和高优先级问题。

- 简历 `/resume`
  - 左侧按模块导航，中间编辑当前模块，右侧为标准 AI 对话流。
  - 支持三类动作：优化当前区块、优化整份简历、按当前岗位做匹配分析。
  - 建议可一键应用到当前区块。

- 记录 `/records`
  - 展示实时助手和模拟面试记录。
  - 记录报告绑定 transcript、提词卡、表达指标与后续动作建议。

## 当前不应再误导描述的点

- 首页不是“猜测式岗位草稿生成器”。
- 本地 fallback 不是“已完成智能解析”。
- 前端本地 `AppState` 不是业务真源。
- 侧栏不会因为进入实时助手而自动收起。

## 持久化真实情况

- 服务端持久化真源：
  - 岗位
  - 简历与证据
  - 当前岗位资料
  - 当前岗位问题
  - 面试记录

- 前端本地仅保留：
  - `serverSnapshotCache`
  - `uiPrefs`
  - `drafts`
